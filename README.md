# Euro Brands — نظام إدارة المخزون والمبيعات

نظام داخلي متكامل لإدارة المخزون والمبيعات لمتجر **Euro Brands** للملابس والأحذية والعطور، بفرعيه في القاهرة:

- 🏬 **حدائق المعادي**
- 🏬 **زهراء المعادي**

الواجهة بالكامل باللغة العربية مع تخطيط من اليمين لليسار (RTL) وخط **Tajawal**، مع دعم الوضع الليلي والنهاري.

---

## المميزات

- **لوحة تحكم (الرئيسية):** إجمالي مبيعات اليوم، مقارنة الفرعين، أكثر منتج مبيعاً، رسوم بيانية (مبيعات يومية + توزيع حسب الفئة)، وأحدث 10 فواتير، مع منتقي فترة زمنية (يومي/أسبوعي/شهري/مخصص).
- **المخزون:** عرض المنتجات في شبكة بطاقات، فلترة بالفرع والفئة والبراند والمقاس، بحث بالاسم أو البراند، وإضافة/تعديل/حذف المنتجات.
- **إضافة/تعديل منتج:** بيانات المنتج + رفع حتى 3 صور (Vercel Blob) + جدول مقاسات وكميات وأسعار لكل فرع.
- **نقطة البيع (الفاتورة):** اختيار الفرع لكل جلسة، بحث سريع، إضافة المقاسات للسلة، تعديل الكميات، بيانات العميل، الخصم (نسبة % أو مبلغ ثابت)، وتأكيد البيعة مع خصم الكميات تلقائياً.
- **التقارير:** تحليل تفصيلي للمبيعات والمنتجات الأكثر مبيعاً والأصناف منخفضة المخزون.
- **سجل الفواتير:** جدول بكل الفواتير مع الفلترة والبحث وعرض تفاصيل كل فاتورة وطباعتها.
- **استيراد الجرد (Excel):** تنزيل قالب Excel وتعبئته ورفعه لتحديث الكميات والأسعار بالجملة مع معاينة قبل التأكيد (SheetJS).
- **الذكاء (Gemini):** تحليلات ذكية لبيانات آخر 30 يوماً والمخزون عبر نموذج Gemini Flash، مع رجوع تلقائي لتحليلات مبنية على قواعد عند غياب المفتاح أو نفاد الحصة.
- **تصدير PDF:** تقرير كامل بالعربية (RTL) عبر jsPDF + autoTable يشمل الفترة والملخص وأكثر المنتجات مبيعاً ومقارنة الفروع وتنبيهات نقص المخزون.

## معاينة كامل الواجهة بدون قاعدة بيانات (وضع المعاينة)

```bash
npm install
npm run dev
```

في **التطوير**، عمل هذا التطبيق افتراضياً ببيانات تجريبية في الذاكرة دون الحاجة لأي خادم أو قاعدة بيانات. تُستبدل كل طلبات الـ API ببيانات تجريبية مباشرة في المتصفح (10 منتجات على الفرعين + عشرات الفواتير على مدى 30 يوماً)، فتعمل لوحة التحكم والرسوم والمخزون ونقطة البيع والتقارير وسجل الفواتير بالكامل.

- يظهر شريط علوي يوضح أنك في وضع المعاينة.
- كل العمليات تعمل فعلياً (تأكيد بيعة، إضافة/تعديل/حذف منتج، رفع صورة) لكن التغييرات **مؤقتة** وتُفقد عند إعادة تحميل الصفحة.

### التحكم بوضع المعاينة عبر `NEXT_PUBLIC_MOCK`

| القيمة | السلوك |
| --- | --- |
| `1` أو `true` | تفعيل البيانات التجريبية (بدون أي طلبات API) |
| `0` أو `false` | استخدام الـ API الحقيقي |
| غير محدد | مفعّل تلقائياً في التطوير، ومعطّل في الإنتاج |

أوامر جاهزة:

```bash
npm run dev:mock     # تطوير ببيانات تجريبية (صريح)
npm run dev:real     # تطوير باستخدام الـ API الحقيقي
npm run build:mock   # بناء نسخة معاينة قابلة للنشر (تعمل بدون قاعدة بيانات)
```

> **نشر معاينة حيّة:** اضبط `NEXT_PUBLIC_MOCK=1` على Vercel (أو استخدم `build:mock`) لمشاركة الواجهة كاملة دون أي قاعدة بيانات.

> **طبقة احتياطية:** حتى بدون وضع معاينة المتصفح، تُرجع مسارات الـ API نفسها بيانات تجريبية تلقائياً عند غياب `DATABASE_URL` (أو بضبط `MOCK_DATA=1`).

## التشغيل محلياً (مع قاعدة بيانات)

### 1) تثبيت الحزم

```bash
npm install
```

### 2) إعداد متغيرات البيئة

انسخ ملف المثال ثم عدّل القيم:

```bash
cp .env.example .env
```

الحد الأدنى المطلوب هو `DATABASE_URL`. لرفع الصور أضف `BLOB_READ_WRITE_TOKEN` (اختياري — يمكن إدخال روابط الصور يدوياً بدونه).

```env
DATABASE_URL="postgresql://USER:PASSWORD@HOST:5432/euro_brands?schema=public"
BLOB_READ_WRITE_TOKEN="vercel_blob_rw_xxx"
```

### 3) تجهيز قاعدة البيانات

```bash
# إنشاء الجداول في قاعدة البيانات
npm run db:push

# تعبئة بيانات تجريبية (5 منتجات + فاتورتان على الفرعين)
npm run db:seed
```

### 4) تشغيل التطوير

```bash
npm run dev
```

ثم افتح المتصفح على: <http://localhost:3000>

## النشر على Vercel

1. ارفع المشروع إلى مستودع Git ثم استورده في Vercel.
2. من **Storage** أنشئ قاعدة **Postgres** و **Blob**؛ ستُضاف متغيرات `DATABASE_URL` و`BLOB_READ_WRITE_TOKEN` تلقائياً.
3. (موصى به) أضف متغير البيئة `TZ=Africa/Cairo` ليكون حساب "مبيعات اليوم" والتجميع اليومي بتوقيت القاهرة.
4. بعد أول نشر، شغّل المزامنة والتعبئة محلياً مع توجيهها لقاعدة الإنتاج:

```bash
npm run db:push
npm run db:seed
```

> **ملاحظة:** أمر البناء على Vercel هو `npm run build` الذي يولّد Prisma Client تلقائياً.

## منطق العمل المهم

- عند تأكيد البيعة تُخصم الكميات تلقائياً من مخزون الفرع المحدد ضمن معاملة (Transaction) واحدة.
- لا يمكن إضافة منتج نفدت كميته إلى الفاتورة (تظهر حالة "نفذت الكمية").
- رقم الفاتورة تصاعدي **عام** (وليس لكل فرع).
- الخصم إمّا نسبة مئوية أو مبلغ ثابت — لا يجتمعان معاً.
- جميع الأسعار بالجنيه المصري (ج.م).
- لا يمكن حذف منتج مرتبط بفواتير سابقة (للحفاظ على دقة سجل المبيعات)؛ يمكن تصفير كمياته بدلاً من ذلك.
